Honeywell Embedded Systems Bluetooth

BlueZ, Yocto, C Language, Embedded Systems, BLE, Cryptography

In spring 2026, I had the opportunity to work with Honeywell on a research project and present my work on developing Bluetooth for a Yocto-based board. I wrote a C Daemon that utilizes BlueZ protocols and encryption over dbus to securely communicate with an iPhone. While BlueZ provides the protocols, the C Daemon handles policy, such as how many devices can connect at once, which MAC addresses are whitelisted, when to use LE vs Legacy, and other security measures such as pairing and discoverability. To ensure secure pairing, the C Daemon requres numeric comparison (shared secret) to mitigate the threat of a MITM attack. After connecting, a key identifying the device is stored, and the Yocto's MAC address regularly rotates. The stored key allows for the previously connected device to still identify and connect to the Yocto. After a secure connection is established, discoverability and pairing is disabled and the C Daemon opens up an encrypted GATT channel. Although the project only uses open-source materials, I am not going to include code here because of the nature of the project.
